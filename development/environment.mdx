## Overview

Environment variables are stored in the `.env` file at the root of your project. This file is created during installation by copying `.env.example`.

<Warning>
**Never commit your `.env` file to version control.** It contains sensitive credentials and configuration specific to your environment.
</Warning>

## Environment File Setup

### Creating Your Environment File

```bash
# Copy example file
cp .env.example .env

# Generate application key
php artisan key:generate
```

### File Structure

The `.env` file uses a simple `KEY=VALUE` format:

```env
APP_NAME="Solare Admin"
APP_ENV=local
DB_DATABASE=solare
```

<Note>
Values containing spaces should be enclosed in quotes. For boolean values, use `true` or `false`.
</Note>

## Application Settings

### Core Application Variables

<ParamField path="APP_NAME" type="string" default="Laravel">
  The name of your application, used in notifications, emails, and UI elements.
  
  ```env
  APP_NAME="Solare Admin"
  ```
</ParamField>

<ParamField path="APP_ENV" type="string" default="production">
  The application environment. Common values: `local`, `development`, `staging`, `production`.
  
  ```env
  APP_ENV=local
  ```
</ParamField>

<ParamField path="APP_KEY" type="string" required>
  Application encryption key (32 characters). Generated automatically by `php artisan key:generate`.
  
  ```env
  APP_KEY=base64:RandomGeneratedKey==
  ```
</ParamField>

<ParamField path="APP_DEBUG" type="boolean" default="false">
  Enable detailed error messages and stack traces. **Must be `false` in production.**
  
  ```env
  APP_DEBUG=true
  ```
</ParamField>

<ParamField path="APP_URL" type="string" default="http://localhost">
  The base URL of your application.
  
  ```env
  APP_URL=http://localhost
  ```
</ParamField>

## Accessing Environment Variables

### In Configuration Files

```php
'name' => env('APP_NAME', 'Default Value'),
```

### In Application Code

```php
// Using env() helper (not recommended in production with cached config)
$appName = env('APP_NAME');

// Using config() helper (recommended)
$appName = config('app.name');
```

<Warning>
When configuration is cached (`php artisan config:cache`), the `env()` function will return `null`. Always use the `config()` helper in your application code.
</Warning>

## Security Best Practices

<Steps>
  <Step title="Never commit .env file">
    Add `.env` to your `.gitignore` file (already done in Laravel).
  </Step>

  <Step title="Use strong encryption keys">
    Always generate a new `APP_KEY` using `php artisan key:generate`.
  </Step>

  <Step title="Disable debug mode in production">
    Set `APP_DEBUG=false` to prevent information leakage.
  </Step>

  <Step title="Use environment-specific values">
    Never use production credentials in development environments.
  </Step>

  <Step title="Restrict file permissions">
    ```bash
    chmod 600 .env
    ```
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Environment variables not loading">
    Clear the configuration cache:
    ```bash
    php artisan config:clear
    ```
  </Accordion>

  <Accordion title="Changes to .env not reflected">
    If you've cached your configuration, you need to recache:
    ```bash
    php artisan config:cache
    ```
  </Accordion>

  <Accordion title="env() returns null in production">
    This is expected when configuration is cached. Use `config()` instead:
    ```php
    // Bad
    $value = env('MY_VAR');
    
    // Good
    $value = config('services.my_service.var');
    ```
  </Accordion>

  <Accordion title="Syntax errors in .env file">
    Ensure:
    - No spaces around the `=` sign
    - Quote values with spaces: `APP_NAME="My App"`
    - No trailing commas or special characters
    - Use `#` for comments on separate lines
  </Accordion>
</AccordionGroup>
