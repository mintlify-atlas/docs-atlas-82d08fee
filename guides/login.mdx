## Overview

Solare Admin View includes a basic login screen template at the root route (`/`). The current implementation showcases Flowbite alert components as examples, providing a foundation for building your authentication interface.

<Note>
  The current login screen is a starter template showing UI component examples. You'll need to implement the actual login form and authentication logic.
</Note>

## Current Implementation

The login screen is located at `resources/views/loginScreen/login.blade.php` and is served by the root route:

```php routes/web.php
Route::get('/', function () {
    return view('loginScreen.login');
});
```

This template demonstrates five Flowbite alert component variants that you can use in your application.

## Alert Components

The login screen showcases Flowbite's alert system with semantic color tokens:

### Brand/Info Alert

For informational messages:

```html
<div class="p-4 mb-4 text-sm text-fg-brand-strong rounded-base bg-brand-softer" 
     role="alert">
  <span class="font-medium">Info alert!</span> Your message here.
</div>
```

- **Background**: `bg-brand-softer` - Soft brand color
- **Text**: `text-fg-brand-strong` - Strong brand text
- **Use case**: General notifications, information

### Danger Alert

For errors and critical warnings:

```html
<div class="p-4 mb-4 text-sm text-fg-danger-strong rounded-base bg-danger-soft" 
     role="alert">
  <span class="font-medium">Danger alert!</span> Error message.
</div>
```

- **Background**: `bg-danger-soft` - Soft red
- **Text**: `text-fg-danger-strong` - Strong red text
- **Use case**: Login errors, validation failures

### Success Alert

For successful operations:

```html
<div class="p-4 mb-4 text-sm text-fg-success-strong rounded-base bg-success-soft" 
     role="alert">
  <span class="font-medium">Success alert!</span> Operation successful.
</div>
```

- **Background**: `bg-success-soft` - Soft green
- **Text**: `text-fg-success-strong` - Strong green text
- **Use case**: Successful login, account created

### Warning Alert

For cautions and warnings:

```html
<div class="p-4 mb-4 text-sm text-fg-warning rounded-base bg-warning-soft" 
     role="alert">
  <span class="font-medium">Warning alert!</span> Caution message.
</div>
```

- **Background**: `bg-warning-soft` - Soft amber/yellow
- **Text**: `text-fg-warning` - Warning text color
- **Use case**: Password expiring, rate limiting

### Neutral/Dark Alert

For neutral system messages:

```html
<div class="p-4 text-sm text-heading rounded-base bg-neutral-secondary-medium" 
     role="alert">
  <span class="font-medium">Dark alert!</span> System message.
</div>
```

- **Background**: `bg-neutral-secondary-medium` - Neutral gray
- **Text**: `text-heading` - Default heading color
- **Use case**: System maintenance, general notices

<Tip>
  These semantic color tokens automatically adapt to both light and dark modes thanks to Flowbite's theme system.
</Tip>

## Building Your Login Form

To create a functional login form, replace the alert examples with a login form. Here's a recommended structure:

```blade login.blade.php
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Login - {{ config('app.name') }}</title>
    
    @vite(['resources/css/app.css', 'resources/js/app.js'])
</head>
<body class="bg-[#FDFDFC] dark:bg-[#0a0a0a]">
    <div class="min-h-screen flex items-center justify-center px-4">
        <div class="max-w-md w-full">
            <!-- Login Card -->
            <div class="bg-white dark:bg-[#161615] rounded-lg shadow-lg p-8 
                        border border-[#e3e3e0] dark:border-[#3E3E3A]">
                <h1 class="text-2xl font-bold text-[#1b1b18] dark:text-[#EDEDEC] mb-6">
                    Sign In
                </h1>

                @if ($errors->any())
                    <div class="mb-4 p-4 text-sm text-fg-danger-strong rounded-base bg-danger-soft" 
                         role="alert">
                        <span class="font-medium">Error!</span> Invalid credentials.
                    </div>
                @endif

                <form method="POST" action="{{ route('login') }}">
                    @csrf
                    
                    <!-- Email Field -->
                    <div class="mb-4">
                        <label for="email" 
                               class="block text-sm font-medium text-[#1b1b18] dark:text-[#EDEDEC] mb-2">
                            Email
                        </label>
                        <input type="email" 
                               id="email" 
                               name="email" 
                               required
                               class="w-full px-4 py-2 border border-[#e3e3e0] dark:border-[#3E3E3A] 
                                      rounded-sm bg-white dark:bg-[#161615] 
                                      text-[#1b1b18] dark:text-[#EDEDEC]
                                      focus:border-[#1915014a] dark:focus:border-[#62605b] 
                                      focus:outline-none">
                    </div>

                    <!-- Password Field -->
                    <div class="mb-6">
                        <label for="password" 
                               class="block text-sm font-medium text-[#1b1b18] dark:text-[#EDEDEC] mb-2">
                            Password
                        </label>
                        <input type="password" 
                               id="password" 
                               name="password" 
                               required
                               class="w-full px-4 py-2 border border-[#e3e3e0] dark:border-[#3E3E3A] 
                                      rounded-sm bg-white dark:bg-[#161615] 
                                      text-[#1b1b18] dark:text-[#EDEDEC]
                                      focus:border-[#1915014a] dark:focus:border-[#62605b] 
                                      focus:outline-none">
                    </div>

                    <!-- Submit Button -->
                    <button type="submit" 
                            class="w-full px-5 py-2.5 bg-[#1b1b18] dark:bg-[#EDEDEC] 
                                   text-white dark:text-[#1b1b18] 
                                   rounded-sm border border-black dark:border-[#3E3E3A]
                                   hover:bg-black dark:hover:bg-white 
                                   font-medium transition-colors">
                        Sign In
                    </button>
                </form>
            </div>
        </div>
    </div>
</body>
</html>
```

## Implementing Authentication

To make the login form functional, you need to:

<Steps>
  <Step title="Install Laravel Breeze or Fortify">
    Use Laravel's authentication packages for login/logout functionality:
    
    ```bash
    composer require laravel/breeze --dev
    php artisan breeze:install blade
    ```
  </Step>
  
  <Step title="Run Migrations">
    Ensure the users table exists:
    
    ```bash
    php artisan migrate
    ```
  </Step>
  
  <Step title="Configure Routes">
    Breeze will add authentication routes automatically, or define them manually:
    
    ```php routes/web.php
    use App\Http\Controllers\Auth\LoginController;
    
    Route::get('/', function () {
        return view('loginScreen.login');
    })->name('login');
    
    Route::post('/login', [LoginController::class, 'authenticate']);
    Route::post('/logout', [LoginController::class, 'logout'])->name('logout');
    ```
  </Step>
  
  <Step title="Create Login Controller">
    Implement authentication logic:
    
    ```php
    php artisan make:controller Auth/LoginController
    ```
  </Step>
</Steps>

## Accessibility

When building your login form, ensure:

- ✅ All form fields have associated `<label>` elements
- ✅ Alerts include `role="alert"` for screen readers
- ✅ Focus states are visible with clear borders
- ✅ Error messages are descriptive and actionable
- ✅ Keyboard navigation works smoothly

## Security Best Practices

<CardGroup cols={2}>
  <Card title="CSRF Protection" icon="shield">
    Always include `@csrf` token in forms to prevent cross-site request forgery attacks.
  </Card>
  
  <Card title="Rate Limiting" icon="gauge">
    Implement rate limiting on login attempts to prevent brute force attacks.
  </Card>
  
  <Card title="HTTPS Only" icon="lock">
    Always use HTTPS in production to encrypt login credentials in transit.
  </Card>
  
  <Card title="Password Hashing" icon="key">
    Laravel automatically hashes passwords using bcrypt. Never store plain text passwords.
  </Card>
</CardGroup>
