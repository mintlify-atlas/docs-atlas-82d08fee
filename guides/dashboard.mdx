## Overview

Solare Admin View is a starter template for building Laravel-based administrative interfaces. The project provides a foundation with authentication scaffolding, Flowbite UI components, and Tailwind CSS v4, but the actual dashboard interface is yours to build.

<Note>
  This is a starter template project. The dashboard interface shown in Laravel's welcome page is not part of the admin application - you'll be building your own dashboard views using the tools provided.
</Note>

## What's Included

The starter template provides:

- **Laravel 12** backend framework
- **User authentication** scaffolding with the User model
- **Flowbite 4** UI component library
- **Tailwind CSS v4** utility-first styling
- **Vite** asset bundling
- **Login screen** at the root route (`/`)

## Current Routes

The application currently has minimal routes defined in `routes/web.php`:

```php routes/web.php
<?php

use Illuminate\Support\Facades\Route;

Route::get('/', function () {
    return view('loginScreen.login');
});
```

You'll need to add your own dashboard routes after authentication is set up.

## Building Your Dashboard

### Step 1: Create Dashboard Route

Add a dashboard route that requires authentication:

```php routes/web.php
use Illuminate\Support\Facades\Route;

Route::get('/', function () {
    return view('loginScreen.login');
});

Route::middleware(['auth'])->group(function () {
    Route::get('/dashboard', function () {
        return view('dashboard');
    })->name('dashboard');
});
```

### Step 2: Create Dashboard View

Create a new Blade template at `resources/views/dashboard.blade.php`:

```blade dashboard.blade.php
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Dashboard - {{ config('app.name') }}</title>
    
    @vite(['resources/css/app.css', 'resources/js/app.js'])
</head>
<body class="bg-[#FDFDFC] dark:bg-[#0a0a0a]">
    <div class="min-h-screen">
        <!-- Header -->
        <header class="bg-white dark:bg-[#161615] border-b border-[#e3e3e0] dark:border-[#3E3E3A]">
            <div class="max-w-7xl mx-auto px-4 py-4">
                <div class="flex justify-between items-center">
                    <h1 class="text-xl font-semibold text-[#1b1b18] dark:text-[#EDEDEC]">
                        Dashboard
                    </h1>
                    <form method="POST" action="{{ route('logout') }}">
                        @csrf
                        <button type="submit" 
                                class="px-4 py-2 text-sm text-[#1b1b18] dark:text-[#EDEDEC] 
                                       border border-[#19140035] dark:border-[#3E3E3A] 
                                       hover:border-[#1915014a] dark:hover:border-[#62605b] 
                                       rounded-sm">
                            Logout
                        </button>
                    </form>
                </div>
            </div>
        </header>

        <!-- Main Content -->
        <main class="max-w-7xl mx-auto px-4 py-8">
            <div class="grid gap-6">
                <!-- Welcome Card -->
                <div class="bg-white dark:bg-[#161615] rounded-lg p-6 
                            border border-[#e3e3e0] dark:border-[#3E3E3A]">
                    <h2 class="text-lg font-semibold text-[#1b1b18] dark:text-[#EDEDEC] mb-2">
                        Welcome, {{ auth()->user()->name }}!
                    </h2>
                    <p class="text-[#706f6c] dark:text-[#A1A09A]">
                        This is your admin dashboard. Start building your interface here.
                    </p>
                </div>

                <!-- Add your dashboard widgets and content here -->
            </div>
        </main>
    </div>
</body>
</html>
```

### Step 3: Add Dashboard Controller

For more complex dashboards, create a dedicated controller:

```bash
php artisan make:controller DashboardController
```

Then update your route:

```php routes/web.php
use App\Http\Controllers\DashboardController;

Route::middleware(['auth'])->group(function () {
    Route::get('/dashboard', [DashboardController::class, 'index'])
        ->name('dashboard');
});
```

## Using Flowbite Components

The template includes Flowbite's alert components, which you can use throughout your dashboard. Here are the alert types available:

### Info/Brand Alerts

```html
<div class="p-4 mb-4 text-sm text-fg-brand-strong rounded-base bg-brand-softer" 
     role="alert">
  <span class="font-medium">Info!</span> General information message.
</div>
```

### Danger Alerts

```html
<div class="p-4 mb-4 text-sm text-fg-danger-strong rounded-base bg-danger-soft" 
     role="alert">
  <span class="font-medium">Error!</span> Something went wrong.
</div>
```

### Success Alerts

```html
<div class="p-4 mb-4 text-sm text-fg-success-strong rounded-base bg-success-soft" 
     role="alert">
  <span class="font-medium">Success!</span> Operation completed.
</div>
```

### Warning Alerts

```html
<div class="p-4 mb-4 text-sm text-fg-warning rounded-base bg-warning-soft" 
     role="alert">
  <span class="font-medium">Warning!</span> Please be careful.
</div>
```

### Neutral Alerts

```html
<div class="p-4 text-sm text-heading rounded-base bg-neutral-secondary-medium" 
     role="alert">
  <span class="font-medium">Notice:</span> Neutral message.
</div>
```

These are the actual alert components defined in `resources/views/loginScreen/login.blade.php` that you can use in your dashboard.

## Design System

The template uses a carefully crafted design system with Tailwind CSS:

### Color Palette

**Light Mode:**
```css
bg-[#FDFDFC]           /* Main background - off-white */
bg-white               /* Card backgrounds */
text-[#1b1b18]        /* Primary text - near black */
text-[#706f6c]        /* Secondary text - gray */
border-[#e3e3e0]      /* Light borders */
```

**Dark Mode:**
```css
bg-[#0a0a0a]          /* Main background - near black */
bg-[#161615]          /* Card backgrounds */
text-[#EDEDEC]        /* Primary text - off-white */
text-[#A1A09A]        /* Secondary text - muted */
border-[#3E3E3A]      /* Dark borders */
```

### Typography

The template uses **Instrument Sans** font family, configured in `resources/css/app.css`:

```css app.css
@theme {
    --font-sans: 'Instrument Sans', ui-sans-serif, system-ui, sans-serif,
        'Apple Color Emoji', 'Segoe UI Emoji', 'Segoe UI Symbol', 'Noto Color Emoji';
}
```

## Adding More Flowbite Components

Flowbite provides many more components beyond alerts. To use them:

1. **Browse components** at [flowbite.com](https://flowbite.com)
2. **Copy component HTML** from the documentation
3. **Paste into your Blade templates**
4. **Customize** with Tailwind classes

<Tip>
  Flowbite's semantic color tokens (like `bg-brand-softer`, `text-fg-danger-strong`) automatically adapt to light and dark mode.
</Tip>

## Authentication Integration

Your dashboard routes should be protected with authentication middleware. The User model and authentication config are already set up:

```php
// In routes/web.php
Route::middleware(['auth'])->group(function () {
    Route::get('/dashboard', [DashboardController::class, 'index']);
    Route::get('/users', [UserController::class, 'index']);
    // Add more protected routes here
});
```

See [Authentication](/features/authentication) for details on the auth system.
