## Installation

<Steps>

<Step title="Clone the Repository">
  Clone the Solare Admin View repository to your local machine:

  ```bash
  git clone <repository-url> solare-admin-view
  cd solare-admin-view
  ```
</Step>

<Step title="Run the Setup Script">
  Solare Admin View includes a convenient Composer script that handles all setup tasks automatically:

  ```bash
  composer setup
  ```

  This single command performs the following operations:
  - Installs all PHP dependencies via `composer install`
  - Copies `.env.example` to `.env` (if not already present)
  - Generates application key with `php artisan key:generate`
  - Runs database migrations with `php artisan migrate --force`
  - Installs Node.js dependencies via `npm install`
  - Builds frontend assets with `npm run build`

  <Note>
  The setup script streamlines the entire installation process. No need to run individual commands!
  </Note>
</Step>

<Step title="Configure Your Database">
  Open the `.env` file and configure your database connection:

  ```bash .env
  DB_CONNECTION=mariadb
  DB_HOST=127.0.0.1
  DB_PORT=3306
  DB_DATABASE=solare
  DB_USERNAME=root
  DB_PASSWORD=your_password
  ```

  Make sure to create the database before running migrations:

  ```bash
  # MySQL/MariaDB
  mysql -u root -p
  CREATE DATABASE solare;
  exit;
  ```

  If you need to run migrations again after database configuration:

  ```bash
  php artisan migrate
  ```
</Step>

<Step title="Start the Development Server">
  Solare Admin View provides a convenient dev script that runs all necessary services concurrently:

  ```bash
  composer dev
  ```

  This command starts three services simultaneously:
  - **Laravel development server** on `http://localhost:8000`
  - **Queue worker** for background job processing
  - **Vite dev server** for hot module replacement (HMR)

  <Tip>
  The output is color-coded for easy identification:
  - 🔵 Blue for the Laravel server
  - 🟣 Purple for the queue worker
  - 🟠 Orange for Vite
  </Tip>
</Step>

<Step title="Access the Login Screen">
  Open your browser and navigate to:

  ```
  http://localhost:8000
  ```

  You should see the Solare Admin View login screen with the Laravel 12 branding!

  <Check>
  If you see the login page, congratulations! Your installation is successful.
  </Check>
</Step>

</Steps>

## Project Configuration

### Application Settings

Key configuration options in your `.env` file:

```bash .env
# Application
APP_NAME=Laravel
APP_ENV=local
APP_DEBUG=true
APP_URL=http://localhost

# Session
SESSION_DRIVER=database
SESSION_LIFETIME=120

# Queue
QUEUE_CONNECTION=database

# Cache
CACHE_STORE=database
```

### Vite Configuration

The Vite configuration is located at `vite.config.js`:

```javascript vite.config.js
import { defineConfig } from 'vite';
import laravel from 'laravel-vite-plugin';
import tailwindcss from '@tailwindcss/vite';

export default defineConfig({
    plugins: [
        laravel({
            input: ['resources/css/app.css', 'resources/js/app.js'],
            refresh: true,
        }),
        tailwindcss(),
    ],
    server: {
        watch: {
            ignored: ['**/storage/framework/views/**'],
        },
    },
});
```

<Note>
The Tailwind CSS v4 Vite plugin is integrated directly, eliminating the need for PostCSS configuration.
</Note>

## Understanding the Routes

The default route configuration in `routes/web.php` is minimal:

```php routes/web.php
use Illuminate\Support\Facades\Route;

Route::get('/', function () {
    return view('loginScreen.login');
});
```

The root URL (`/`) displays the login view located at `resources/views/loginScreen/login.blade.php`.

## Frontend Asset Structure

### CSS Entry Point

The main CSS file uses Tailwind CSS v4's new syntax:

```css resources/css/app.css
@import "tailwindcss";
@import "flowbite/src/themes/default";

@source "../../resources/views/**/*.blade.php";
@source "../../resources/js/**/*.js";
@source "../../vendor/laravel/framework/src/Illuminate/Pagination/resources/views/*.blade.php";

@plugin "flowbite/plugin";

@theme {
    --font-sans: 'Instrument Sans', ui-sans-serif, system-ui, sans-serif;
}
```

### JavaScript Entry Point

The main JavaScript file bootstraps Flowbite:

```javascript resources/js/app.js
import './bootstrap';
import 'flowbite'; // Activates modals, dropdowns, etc.
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Database Connection Error">
    If you encounter database connection errors:
    
    1. Verify your database credentials in `.env`
    2. Ensure the database exists: `CREATE DATABASE solare;`
    3. Check that your database server is running
    4. Run migrations: `php artisan migrate`
  </Accordion>

  <Accordion title="Vite Build Errors">
    If Vite fails to build assets:
    
    1. Clear npm cache: `npm cache clean --force`
    2. Delete `node_modules`: `rm -rf node_modules`
    3. Reinstall dependencies: `npm install`
    4. Rebuild assets: `npm run build`
  </Accordion>

  <Accordion title="Permission Issues">
    Laravel requires write permissions for certain directories:
    
    ```bash
    chmod -R 775 storage bootstrap/cache
    ```
  </Accordion>

  <Accordion title="Port Already in Use">
    If port 8000 is already in use, specify a different port:
    
    ```bash
    php artisan serve --port=8080
    ```
  </Accordion>
</AccordionGroup>
