## Overview

Solare Admin View uses Flowbite, a component library built on top of Tailwind CSS, to create beautiful and accessible user interfaces. Flowbite provides pre-designed components that follow best practices for accessibility and responsive design.

## Flowbite Integration

Flowbite is integrated into the project through the CSS configuration at `resources/css/app.css:2`:

```css
@import "tailwindcss";
@import "flowbite/src/themes/default";

@source "../../resources/views/**/*.blade.php";
@source "../../resources/js/**/*.js";
@source "../../vendor/laravel/framework/src/Illuminate/Pagination/resources/views/*.blade.php";

@plugin "flowbite/plugin";

@theme {
    --font-sans: 'Instrument Sans', ui-sans-serif, system-ui, sans-serif, 'Apple Color Emoji', 'Segoe UI Emoji',
        'Segoe UI Symbol', 'Noto Color Emoji';
}
```

<Note>
The configuration imports Flowbite's default theme and registers the Flowbite plugin, making all components available throughout the application.
</Note>

## Component Usage Guidelines

### When to Use Each Alert Type

<Accordion title="Info Alerts">
  Use info alerts for neutral, informational messages that don't require immediate action. Examples:
  - Feature announcements
  - Helpful tips
  - General notifications
</Accordion>

<Accordion title="Danger Alerts">
  Use danger alerts for errors and critical issues that require attention. Examples:
  - Form validation errors
  - Failed operations
  - Security warnings
</Accordion>

<Accordion title="Success Alerts">
  Use success alerts to confirm successful operations. Examples:
  - Successful form submissions
  - Completed actions
  - Saved changes
</Accordion>

<Accordion title="Warning Alerts">
  Use warning alerts for potentially problematic situations. Examples:
  - Upcoming deadlines
  - Resource limitations
  - Deprecated features
</Accordion>

<Accordion title="Dark Alerts">
  Use dark alerts for secondary information with reduced emphasis. Examples:
  - Contextual help text
  - Optional information
  - Supplementary details
</Accordion>

## Blade Template Integration

The login view at `resources/views/loginScreen/login.blade.php:11` uses Blade's templating features:

```blade
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Mi App con Flowbite</title>
    
    @vite(['resources/css/app.css', 'resources/js/app.js'])
</head>
<body>
    @yield('content')
    
    <!-- Alert components -->
</body>
</html>
```

<Note>
The `@vite` directive automatically includes the compiled CSS and JavaScript assets, ensuring Flowbite components are properly styled and functional.
</Note>

## Dynamic Alerts in Laravel

You can dynamically render alerts based on session data:

```blade
@if (session('success'))
    <div class="p-4 mb-4 text-sm text-fg-success-strong rounded-base bg-success-soft" role="alert">
        <span class="font-medium">Success!</span> {{ session('success') }}
    </div>
@endif

@if (session('error'))
    <div class="p-4 mb-4 text-sm text-fg-danger-strong rounded-base bg-danger-soft" role="alert">
        <span class="font-medium">Error!</span> {{ session('error') }}
    </div>
@endif
```

## Customization

You can customize Flowbite components by extending the theme configuration:

```css
@theme {
    /* Custom colors */
    --color-brand-primary: #your-color;
    
    /* Custom fonts */
    --font-sans: 'Your Font', sans-serif;
    
    /* Custom spacing */
    --spacing-base: 1rem;
}
```

## Best Practices

<Steps>
  <Step title="Use Semantic Colors">
    Choose alert types based on message semantics, not visual preference.
  </Step>
  
  <Step title="Keep Messages Concise">
    Alert messages should be brief and actionable.
  </Step>
  
  <Step title="Include Actions">
    When appropriate, provide buttons or links for users to resolve issues.
  </Step>
  
  <Step title="Test Accessibility">
    Always test components with screen readers and keyboard navigation.
  </Step>
</Steps>

<Warning>
Avoid using too many alerts on a single page. Multiple alerts can overwhelm users and reduce their effectiveness.
</Warning>
